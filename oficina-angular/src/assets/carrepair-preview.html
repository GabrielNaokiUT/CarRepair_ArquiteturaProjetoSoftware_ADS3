Sobre o projeto

O CarRepair é um sistema acadêmico de gestão de oficina mecânica desenvolvido na disciplina de Arquitetura e Projeto de Software no SENAI FATESG. O objetivo principal não é apenas construir o sistema, mas demonstrar a evolução arquitetural de um projeto real.

Objetivo acadêmico
O que o sistema faz

O CarRepair gerencia o ciclo completo de atendimento de uma oficina mecânica: cadastro de clientes e seus veículos, cadastro de mecânicos responsáveis pela execução técnica, cadastro de usuários operadores do sistema, abertura e acompanhamento de Ordens de Serviço e registro de serviços executados e peças aplicadas em cada OS.

Arquitetura em camadas

Controller → Service → Validation → Repository → Model. Fluxo obrigatório em todo domínio, com interface e implementação separadas por camada.

Reuso com Generics

Classes genéricas no pacote core eliminam duplicação de CRUD. Novos domínios herdam o comportamento completo sem reescrever nenhuma operação básica.

API REST desacoplada

DTOs garantem que a API nunca expõe entidades JPA diretamente. Contrato de erros padronizado via GlobalExceptionHandler com @RestControllerAdvice.
